# Open-Agent-Teams 路由规则（OpenClaw 判型参考）

> 规范性说明见 [设计规格 §3](../specs/2026-04-26-open-agent-teams-design.md)。本表用于人工或插件侧路由配置，非可执行代码。

## 1. 默认走 OpenClaw 内核（自执行）

满足以下特征时，优先由内核直接调度工具链完成，**不**调用 Hermes：

| 类别 | 示例 |
| --- | --- |
| 文件与系统 | 读写作业目录、批量重命名、本地脚本执行（在权限允许范围内） |
| 网络与信息检索 | 通用网页抓取、公开资料汇总、与个人账户无关的 fact 查询 |
| 办公与自动化 | 表格/文档格式转换、模板化报表、定时任务编排 |
| 运维与监控 | 日志拉取、健康检查、与「个人心智」无关的排障 |

## 2. 下发 Hermes 垂类实例

满足以下**任一**时，应将请求路由到 **已注册且标签匹配** 的 Hermes 实例（见 [hermes-instances.template.yaml](./hermes-instances.template.yaml)）：

| 类别 | 示例 |
| --- | --- |
| 个人习惯与偏好 | 长期饮食/作息/消费偏好、重复出现的个人决策模式 |
| 专属项目上下文 | 某仓库的编码风格、分支策略、内部术语表 |
| 长期跟踪 | 多会话延续的 OKR、习惯养成、研究主题演进 |
| 垂类深度 | 强领域写作风格、行业观点沉淀、个性化研究复盘 |

## 3. 灰区与人工策略

- **可拆任务**：先内核完成资料收集，再将「个性化综合」子任务下发 Hermes，由内核合并输出。  
- **不确定**：默认走内核并明确告知用户「未命中垂类路由」；或反问需要绑定的 Hermes `label`/`tags`。  
- **安全**：涉及敏感写操作前，由内核权限模型把关；Hermes 仅接收完成任务所需的最小上下文。

## 4. 与 OpenClaw 配置的对应关系

在 OpenClaw 侧将上表落实为：

- 插件/函数中的 **标签匹配规则**（关键词、意图分类器或 LLM 判型输出 → `instance_id`）。  
- **超时与重试**：建议每实例单独 `timeoutMs`；失败时内核兜底（见规格 §5）。
